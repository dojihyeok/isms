# ISMS-P 통합 컴플라이언스 대시보드

ISMS-P 운영 업무와 ISO/IEC 27001:2022 SoA를 한 화면에서 탐색하는 React 기반 프로토타입입니다.

## 주요 기능

- ISMS-P 현황 대시보드와 통제 매트릭스
- 보안 업무, 체크리스트, 결재 워크플로우 시뮬레이션
- 증적 보관소와 외부 심사원 Q&A
- 감사 로그 조회
- ISO 27001 SoA 및 ISMS-P 통제항목 매핑·Gap 분석
- CISO, 실무자, 외부 심사원 역할별 화면 전환

## 실행 방법

```bash
npm install
npm run dev
```

품질 검증과 배포용 빌드:

```bash
npm run check
```

개별 실행이 필요한 경우:

```bash
npm run lint
npm run build
npm run preview
```

## 기술 구성

- React 19
- TypeScript
- Vite
- Lucide React

## 데이터와 보안 범위

현재 버전은 프론트엔드 프로토타입입니다. 업무, 증적 메타데이터, Q&A와 감사 로그는 브라우저 LocalStorage에 저장됩니다. 파일 업로드·다운로드, 해시, 암호화 및 역할 기반 접근통제는 실제 서버 보안 기능이 아닌 UI 시뮬레이션입니다. 운영 환경에 적용하려면 인증·인가, API, 데이터베이스, 파일 저장소와 서버 측 감사 로그를 별도로 구현해야 합니다.

참조 데이터는 `refer/`에, 애플리케이션용 ISO/ISMS 매핑 데이터는 `src/isoData.ts`에 있습니다.

## 현재 개발 상태

프론트엔드 프로토타입 범위의 주요 화면과 상호작용은 구현 완료 상태입니다.

- 밝은 단색 기반 반응형 UI
- 상태별 KPI와 영역별 그래프
- 실제 마감일 기반 월간 캘린더
- 태스크 수행, 체크리스트, 증적 등록 및 결재 시뮬레이션
- 외부 심사원 Q&A와 감사 로그
- ISO 27001 SoA 및 ISMS-P Gap 분석
- 보안조직 강화안, 부서별 R&R, 인증 로드맵 및 규정·지침 개정안
- 전자금융감독규정·ISMS-P·ISO 27001·내부지침 기반 운영활동 증적 보관소
- 손상된 브라우저 저장 데이터 자동 복구
- TypeScript 빌드와 ESLint 품질 검사 통과

주요 화면 컴포넌트는 `src/components/`, 공통 도메인 타입은 `src/models/`, 저장소 훅은 `src/hooks/`에 분리되어 있습니다.

## 운영 전 추가 구축 항목

실제 운영 서비스로 전환하려면 다음 서버 기능이 필요합니다.

- 사용자 인증과 서버 측 역할 기반 권한검사
- API 및 데이터베이스 영속화
- 증적 파일 암호화 저장소와 실제 SHA-256 검증
- 변경 불가능한 서버 감사 로그
- 백업·복구, 모니터링, CI/CD 및 자동화 테스트
